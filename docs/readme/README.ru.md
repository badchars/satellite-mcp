## Обзор

**satellite-mcp** -- это MCP-сервер полного спектра GEOINT (геопространственная разведка), предоставляющий вашему ИИ-агенту 171 инструмент в 27 категориях через [Model Context Protocol](https://modelcontextprotocol.io). От спутниковых снимков и обнаружения пожаров до отслеживания воздушных судов, анализа морского трафика, военной разведки и мониторинга конфликтов -- агент получает доступ ко всему в одном диалоге.

## Быстрый старт

### Вариант 1: npx (без установки)

```bash
npx satellite-mcp
```

### Вариант 2: Глобальная установка

```bash
npm install -g satellite-mcp
satellite-mcp
```

### Подключение к ИИ-агенту

<details open>
<summary><b>Claude Code</b></summary>

```bash
claude mcp add satellite-mcp -- npx satellite-mcp
```

</details>

<details>
<summary><b>Claude Desktop</b></summary>

Добавьте в `~/Library/Application Support/Claude/claude_desktop_config.json`:

```json
{
  "mcpServers": {
    "satellite": {
      "command": "npx",
      "args": ["-y", "satellite-mcp"]
    }
  }
}
```

</details>

<details>
<summary><b>Cursor / Windsurf / другие MCP-клиенты</b></summary>

Тот же формат конфигурации JSON. Укажите команду на `npx satellite-mcp` или путь к локальной установке.

</details>

## API-ключи

Многие инструменты работают без API-ключей, используя свободно доступные источники данных. Премиум-источники данных требуют отдельных API-ключей, настраиваемых как переменные окружения.

```bash
# Примеры необязательных API-ключей
export SENTINEL_API_KEY=your-key
export NASA_API_KEY=your-key
export PLANET_API_KEY=your-key
export OPENWEATHER_API_KEY=your-key
```

Все ключи необязательны. Сервер деградирует корректно и возвращает описательные сообщения об ошибках, когда ключ отсутствует для данного инструмента.

## Архитектура

```
src/
  index.ts                # Точка входа CLI и настройка MCP-сервера
  protocol/
    tools.ts              # Реестр инструментов -- все 171 инструмент собраны здесь
  providers/
    sentinel/             # Инструменты снимков Sentinel-2 (8)
    landsat/              # Инструменты снимков Landsat (7)
    firms/                # Инструменты пожаров NASA FIRMS (6)
    nasa-earth/           # Наблюдение Земли NASA (7)
    nightlights/          # Инструменты ночного освещения VIIRS (5)
    planet/               # Инструменты Planet Labs (4)
    aircraft/             # Инструменты авиационной разведки (10)
    maritime/             # Инструменты морской разведки (8)
    military/             # Военные и оборонные инструменты (8)
    space/                # Космические и орбитальные инструменты (6)
    conflict/             # Инструменты конфликтов и событий (6)
    environmental/        # Экологические инструменты (6)
    infrastructure/       # Инструменты критической инфраструктуры (6)
    sanctions/            # Инструменты санкций (5)
    terrain/              # Инструменты рельефа (6)
    cyber-geo/            # Инструменты кибер-гео (6)
    population/           # Инструменты населения (5)
    agriculture/          # Сельскохозяйственные инструменты (5)
    humanitarian/         # Гуманитарные инструменты (5)
    ocean/                # Океанические инструменты (5)
    trade/                # Торговые инструменты (5)
    seismic/              # Сейсмические инструменты (5)
    osm/                  # Инструменты OpenStreetMap (8)
    geocoding/            # Инструменты геокодирования (8)
    spectral/             # Инструменты спектрального анализа (8)
    change-detection/     # Инструменты обнаружения изменений (6)
    weather/              # Инструменты погоды (6)
```

**Проектные решения:**

- **27 провайдеров, 1 сервер** &mdash; Каждый источник данных -- независимый модуль. Агент выбирает нужные инструменты на основе запроса.
- **Ограничители запросов для каждого провайдера** &mdash; Каждый источник данных имеет собственный экземпляр `RateLimiter`, откалиброванный под лимиты соответствующего API.
- **TTL-кеширование** &mdash; Спутниковые данные, погода и геополитические ленты кешируются для предотвращения избыточных вызовов API.
- **Корректная деградация** &mdash; Отсутствующие API-ключи не приводят к сбою сервера. Инструменты возвращают описательные сообщения об ошибках.
- **Без настройки** &mdash; `npx satellite-mcp` &mdash; одна команда, никакой конфигурации.
